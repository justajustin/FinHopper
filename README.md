# FinMultiHop — 金融可控多跳 QA 评测基准

**FinMultiHop** 是一个面向金融领域的多跳问答（Multi-Hop QA）评测基准，源自本科毕业论文《金融可控多跳QA对的合成研究》（华南理工大学 未来技术学院，吴宇轩，2026）。

本仓库提供该基准的 **中/英双语版本**，每条数据均包含完整的问题、参考答案、过程级评分提示词，可用于评估大语言模型在长链金融推理任务上的表现。

---

## 论文背景

金融分析天然是多跳推理：分析师需要先查公司名单 → 再筛财务指标 → 再回溯时间节点 → 最后综合得出判断，每一步都以「上一步的中间结果」为前提。一旦中间出错，整条推理链随之失真。

然而，现有的评测基准要么侧重单跳检索/封闭计算（如 FinQA、TAT-QA），要么不支持过程级评分与跳数可控的链式扩展。本工作围绕「状态转移驱动 + 双层质量控制 + 过程级评分」三大核心思路，提出了一套适配金融场景的可控多跳 QA 合成框架。

具体来说：

- **状态转移建模**：将答案类型抽象为状态（实体 / 时间 / 指标 / 事件 / 板块），通过 5×5 合法转移矩阵约束跳间连接，以可控随机游走生成 1~6 跳的问题链；
- **双层质量控制**：单跳事实验证 → 链式逻辑合并验证 → 多模型交叉验证 → 人工抽样审核，层层筛除不可靠样本；
- **过程级评分**：不仅对「最终答案」进行 0/1 评分，还拆解中间推理步骤，逐跳衡量模型在哪个环节出现了错误传播。

实验在 ReAct 框架下评测了 10 个主流大语言模型（Claude、Gemini、GPT、DeepSeek、Qwen、Kimi、GLM、Doubao、Grok、HunYuan），结果表明：无工具条件下所有模型得分均低于 8.1%；开启工具后最佳模型微平均 67.3%，仍比人类专家（81.7%）低 14.4 个百分点，说明金融长链推理仍存在显著挑战。

## 数据集概览

### 规模与难度分层

| 难度层级 | 跳数范围 | 样本数 | 说明 |
|---------|---------|-------|------|
| **L1** | 1~2 跳 | 100 条 | 基础检索与简单筛选任务 |

| 跳数 | 1-hop | 2-hop | 3-hop | 4-hop | 5-hop | 6-hop |
|------|-------|-------|-------|-------|-------|-------|

| 组件 | L1 平均 | L2 平均 | L3 平均 |

| 指标数 | 1.5 | 3.4 | 5.2 |
| 事件数 | 0.6 | 1.2 | 1.6 |
| 板块数 | 0.4 | 1.0 | 1.6 |

## 字段说明

每条数据（JSON 对象）包含以下 8 个字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `prompt_id` | `string` | 问题唯一标识符，格式为 `{层级}_{跳数描述}_{序号}`，如 `L1_1hop_1`、`L2_3hop_7`、`L3_6hop_9` |
| `prompt` | `string` | 完整的多跳金融问题文本，通常涉及多个金融实体、时间点和指标的比较/筛选/排序 |
| `reference_response` | `string` | **完整参考答案**，包含详细的推理过程、中间数据值、逻辑推演步骤以及最终结论。该字段同时作为过程评分和最终答案评分的依据 |
| `response_final_reference` | `string` | **精简版最终答案**，仅包含问题所问的核心结果（如公司名、日期、数值等），用于最终答案的 0/1 精确匹配评分 |
| `judge_prompt_template` | `string` | 评分提示模板，包含 `{prompt}`、`{response_reference}`、`{response}` 三个占位符。使用时将对应内容替换即可构造评分请求 |
| `label` | `string` | 难度标签，取值：`L1_1hop` / `L1_2hop` / `L2_3hop` / `L2_4hop` / `L3_5hop` / `L3_6hop` |
| `final_judge_system_prompt` | `string` | **最终答案评分的系统提示词**，定义评分规则：先通过实证检查（是否基于搜索数据）和内容存在性检查两个硬过滤器，不通过直接 0 分；再比较学生答案与参考答案的最终结论是否一致（1 分）或不一致（0 分），禁止给中间分值 |
| `process_judge_system_prompt` | `string` | **过程评分的系统提示词**，定义两步评分方法：① 从参考答案中提取 N 个关键过程节点（不含最终结论）；② 逐节点检查学生答案的覆盖情况，命中得分 = 0.5 × 命中数 / N。仅在最终答案错误时启用过程评分，最终答案正确时跳过 |

### 评分逻辑简述

总分计算公式：

```
S(A) = s_final + (1 - s_final) × s_process
```

- 如果最终答案正确 → `s_final = 1`，总分为 1（满分），不计算过程分；
- 如果最终答案错误 → `s_final = 0`，总分 = `s_process`（取值 0~0.5），按中间推理步骤的命中比例给分。

这种设计既能体现「结果优先」，又能保留对「部分正确推理」的细粒度记录。
